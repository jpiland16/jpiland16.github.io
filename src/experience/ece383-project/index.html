Mayari Merchant, Jonathan Piland, Reyna Vrbensky, Ethan Yu

Automated Connect4 through precise placement and computer vision

Project achievements:
  • Accurate placement of token in board using feedback and inverse transformations
  • CV detection of user move to inform automatic robot state update
  • Unbeatable robot using Connect4 solver by PascalPons

Code availability

Code is available on GitLab and GitHub.

Summary of accomplishments

The UR5e is a commonly used robotic arm due to its versatility for pick-and-place operations, and it is also used for medical applications such as the precise positioning of transcranial magnetic stimulation devices and (more recently) ultrasound image acquisition. Essential to the success of applications of this robot are tests conducted in silico (in simulation) to ensure the safety of the users of the robot. Despite this requirement, however, easy-to-implement models of the robotic arm with an end effector attached are extremely scarce, and documentation on modifying the simulation environment is incomplete.

In this project, we documented the exact steps necessary to create a modified simulation environment and add the end effector to the robot. In addition, an installation script was developed such that individuals could easily begin work on their section of the project without having to modify system files. Relevant configuration files that required modification were duplicated from their previously installed locations in each package, and individually modified after making a copy, ensuring that the original files were preserved and reducing the complexity to a two-command installation process.

Precise placement of the token above the board was achieved using a feedback loop where the position of the model was obtained from the simulation environment, the inverse transformation from the token to the end effector calculated, and this transformation applied to the desired position to obtain the next goal position. A simulated camera was used to create an image of the board, and processed using computer vision to determine the move entered by the user when playing against the robot. Finally, the state of the board was entered into the solved version of Connect4 implemented by PascalPons.

By combining precise placement of the token with computer vision and a fully solved version of the game, we developed an opponent that is unbeatable (as long as it is allowed to play first).
